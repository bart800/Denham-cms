## 1. Overview

After a client signs the ATR, there are two parallel tracks:
1. **Request documents from the client** â€” correspondence, estimates, photos, policy
2. **Send the Letter of Representation (LOR) to the insurance company** â€” this triggers the carrier to communicate through us and send their file

Both should happen as quickly as possible after signup.

## 4. LOR Follow-Up

Set a task for **2 days** after the LOR is sent.

### Kentucky Claims Follow-Up

Reply to the original LOR email:

> Please acknowledge receipt and provide the requested information, failure to do so will be used as further evidence of bad faith.
>
> "Failing to acknowledge and act reasonably promptly upon communications with respect to claims arising under insurance policies" is a breach of KRS 304.12-230(2) Unfair Claims Settlement Practices.
>
> "Failing to promptly provide a reasonable explanation of the basis in the insurance policy in relation to the facts or applicable law for denial of a claim or for the offer of a compromise settlement;" is a breach of KRS 304.12-230(14) Unfair Claims Settlement Practices.

### All Other States Follow-Up

> Following up on this. Please acknowledge receipt and provide the requested information.

âš ï¸ **WE DO NOT FOLLOW UP ON CONTRACTOR NEGLIGENCE LORs.**

Save anything the adjuster sends to the CMS and OneDrive backup.

>

> Thanks!

### From Contractors

> Hi [Contractor Name],
>
> This is [Name] with the Denham Property and Injury Law Firm. Our firm has retained [client name], who we understand has used your services for [insert type of work done]. >
> Thank you,
> [Signature]

- If you don't have direct contact info, call the company to find the right person
- Upload all documents to corresponding folders and update the team

## 6. Document Naming Convention

### Format: `M.D.YY [Document Name]`

**Examples:**
```
2.14.25 Demand
2.14.25 Demand Response Denham v. State Farm
9.23.25 - AOD (entered) - J Smith
8.26.25 SF Letter re Settlement
9.1.25 LOR Smith
Certified Policy from [SF, Progressive, Allstate, etc.]
7.2.25 Allstate Letter re Damages and Investigation
4.2.25 IRS Estimate
3.3.25 Progressive Denial Letter
2.2.25 Kate Dick's Engineering Report
```

### Standard Abbreviations

| Abbreviation | Meaning |
|-------------|---------|
| NOD | Notice of Deposition |
| Depo | Deposition |
| RFA | Request for Admission |
| INT | Interrogatories |
| RFPD | Request for Production of Documents |
| LOR | Letter of Representation |
| POL | Proof of Loss |
| POA | Power of Attorney |
| PTC | Pre-Trial Conference |
| Def | Defendant |
| Plf | Plaintiff |

### Rules
- Save **drafts** as Word docs with initials at the end (e.g., "Demand Draft AD")
- Save **final versions** as PDFs
- Add **Plf v. Defendant** at the end of incoming responses
- Format: `[Date] - [Document name] - [client name first initial last name]`

## 9. Scanning & Filing Procedures

### 9.1 Scanning Physical Documents

1. Open the **Windows Scan** app
2. Select the appropriate source:
   - **Stack of documents** â†’ Feeder
   - **Single-sided document or check** â†’ Flatbed
3. Set File Type to **PDF**
4. If double-sided, enable **"Scan both sides"**
5. Save file to the corresponding client folder
6. Remove any staples before scanning
7. Ensure documents are in correct order
8. Click **Scan**

### 9.2 Processing Scanned Documents in Adobe

1. Open the PDF in Adobe Acrobat
2. Under "All tools" â†’ select **"Scan & OCR"** â†’ **Recognize Text** â†’ In this file
   - This makes the PDF searchable
3. If the file needs to be split (e.g., a stack from the insurance company containing policy, estimates, etc.):
   - Go to **Edit** â†’ **Organize Pages**
   - Select the extract icon â†’ **"Extract pages as separate files"**
   - Rename files according to naming convention
   - Save to corresponding folders

### 9.3 Physical Filing

1. Ensure documents are in correct order
2. Re-staple if necessary
3. Place each document type in a labeled manila folder
4. Place in the corresponding hanging folder in the file cabinet
5. If no hanging folder exists, create one: **Last Name, First Name** (alphabetical order)
